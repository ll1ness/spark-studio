# Руководство по тестированию адаптивности TechOn UI

## Запуск локального сервера

Сервер уже запущен на `http://127.0.0.1:8080`

Для доступа с мобильных устройств используйте:
- `http://192.168.1.XXX:8080` (ваш локальный IP адрес)

## Точки разрыва (Breakpoints)

Сайт использует mobile-first подход со следующими точками адаптации:

| Breakpoint | Ширина | Устройства | Изменения |
|------------|--------|------------|-----------|
| Base | 320px+ | Мобильные (малые) | Базовые стили, хамбургер-меню |
| 480px | 480px+ | Мобильные (средние) | Увеличенные отступы, кнопки в ряд |
| 768px | 768px+ | Планшеты, маленькие ноутбуки | Горизонтальное меню, 2-3 колонки |
| 1024px | 1024px+ | Ноутбуки, десктопы | Максимальные отступы, 3 колонки |
| 1200px | 1200px+ | Большие десктопы | Центрирование контента |
| 1440px | 1440px+ | Широкие экраны | Увеличенная типографика |

## Что проверить на каждом устройстве

### 1. Мобильные устройства (320px - 767px)
- [ ] Хамбургер-меню отображается и кликабельно
- [ ] При клике на хамбургер открывается боковое меню
- [ ] Меню закрывается при клике на ссылку
- [ ] Меню закрывается при клике вне его области
- [ ] Меню закрывается по клавише Escape
- [ ] Все кнопки и ссылки минимум 44x44px
- [ ] Нет горизонтального скролла
- [ ] Текст читабелен без зума
- [ ] Изображения не выходят за границы
- [ ] Формы удобны для заполнения
- [ ] Прокрутка плавная

### 2. Планшеты (768px - 1023px)
- [ ] Горизонтальное меню видно
- [ ] Хамбургер скрыт
- [ ] Сетка компонентов: 2 колонки
- [ ] Hero-секция: 2 колонки
- [ ] Все элементы масштабируются корректно
- [ ] Отсутствие горизонтального скролла

### 3. Ноутбуки (1024px - 1199px)
- [ ] Максимальная ширина контента 1200px
- [ ] Сетка компонентов: 3 колонки
- [ ] Hero-секция: 2 колонки с соотношением 2:1.4
- [ ] Все отступы увеличены
- [ ] Типографика оптимальна

### 4. Десктопы (1200px+)
- [ ] Контент центрирован
- [ ] Все элементы на месте
- [ ] Нет излишних отступов

## Критерии успешной адаптивности

### ✅ Производительность
- [ ] Страница загружается быстро на мобильных сетях
- [ ] Изображения lazy-loaded
- [ ] Минимальные перерисовки при скролле
- [ ] Плавные анимации (60 FPS)

### ✅ Доступность (Accessibility)
- [ ] Все интерактивные элементы доступны с клавиатуры
- [ ] Фокус-стайлы видны и понятны
- [ ] ARIA-атрибуты корректны
- [ ] Skip-link работает
- [ ] Контрастность текста достаточна
- [ ] Поддержка prefers-reduced-motion

### ✅ Сенсорное управление
- [ ] Минимальный размер касания: 44x44px
- [ ] Отступы между элементами достаточные
- [ ] Нет accidental taps
- [ ] Кнопки имеют обратную связь при нажатии

### ✅ Читаемость
- [ ] Текст не требует горизонтального скролла
- [ ] Размер шрифта адаптируется под экран
- [ ] Межстрочный интервал достаточный
- [ ] Контрастность соответствует WCAG AA

### ✅ Навигация
- [ ] Меню работает на всех устройствах
- [ ] Активный пункт меню подсвечивается
- [ ] Плавный скролл к якорям
- [ ] Кнопка "Наверх" появляется при прокрутке
- [ ] Хедер sticky при прокрутке

## Инструменты для тестирования

### 1. Chrome DevTools
- Откройте DevTools (F12)
- Включите Device Toolbar (Ctrl+Shift+M)
- Выберите устройства:
  - iPhone SE (375px)
  - iPhone 12 Pro (390px)
  - iPad (768px)
  - iPad Pro (1024px)
  - Laptop (1366px)
  - Desktop (1920px)

### 2. Firefox Responsive Design Mode
- Ctrl+Shift+M
- Тестирование на реальных размерах устройств

### 3. Safari Responsive Mode
- Develop → Enter Responsive Design Mode

### 4. Реальные устройства
- Откройте сайт на реальном смартфоне/планшете
- Проверьте touch-взаимодействие
- Протестируйте в разных ориентациях

## Известные проблемы и решения

### Горизонтальный скролл
Если появляется горизонтальный скролл:
1. Проверьте `overflow-x: hidden` на body и html
2. Убедитесь, что все элементы имеют `max-width: 100%`
3. Проверьте, нет ли элементов с фиксированной шириной

### Меню не закрывается
Убедитесь, что JavaScript загрузился:
- Проверьте консоль на ошибки
- Убедитесь, что `main.js` подключен корректно

### Touch targets слишком маленькие
Все кнопки и ссылки имеют `min-height: 44px` и `min-width: 44px`.

## Проверка Lighthouse

Запустите Lighthouse в Chrome DevTools:
1. DevTools → Lighthouse
2. Выберите категории:
   - Performance
   - Accessibility
   - Best Practices
   - SEO
3. Проверьте, что нет ошибок по:
   - Touch targets не достаточно велики
   - Content not sized appropriately for viewport
   - Horizontal scrolling
